# Lightning Web Components

Components in this folder should be small, record-driven, and Experience Cloud friendly.

## Component Contracts

Each component should declare:

- Public properties exposed to Experience Builder.
- Salesforce objects and fields it depends on.
- Empty state behavior.
- Accessibility requirements.
- Content manager controls.

## Initial Components

| Component | Purpose |
| --- | --- |
| `portalHomeShell` | Home page layout shell for personalized alumni content regions. |
| `portalContentCard` | Reusable card for Salesforce-managed content records. |
| `portalContentList` | Record-driven list of published portal content. |
| `portalBrandHeader` | UChicago-aware header shell pending final brand asset approval. |
| `portalNavigation` | UChicago-inspired utility and primary navigation for portal pages. |
